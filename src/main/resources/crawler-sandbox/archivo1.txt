El método de instalación es diferente del propuesto por el profesor para evitar tener que lidiar con errores y fallos inoportunos, fruto de la experiencia previa con este mismo software y la multitud de problemas que me ha causado en el pasado. 
 
En lugar de utilizar el instalador para Windows de Oracle Database 18c Express Edition he optado por virtualizar este SGBD en un contenedor Docker basándome en la guía oficial de Oracle que se puede encontrar en este enlace. Un contenedor Docker en esencia es una máquina virtual muy ligera basada en Linux y aislada del sistema operativo anfitrión que permite empaquetar una aplicación y sus dependencias como un "ejecutable" completo. El intérprete de este ejecutable es el software Docker, que permite crear contenedores a partir de una imagen Docker, así como configurarlos para exponer al sistema operativo anfitrión medios para interactuar con el mismo. Como estos contenedores están basados en Linux, en Windows existen dos back-ends de virtualización con los que puede funcionar Docker: contenedores de Hyper-V y WSL2 (NOTA: Hyper-V es una característica exclusiva de la versión Pro de Windows). Yo he optado por emplear WSL2 porque ya dispongo de ello para otros propósitos.
